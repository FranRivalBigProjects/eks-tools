Frequently Asked Questions

How much does an AWS EKS cluster cost?

EKS charges $0.10 per hour per cluster ($72/month) for the control plane, plus the cost of the worker nodes. For managed node groups, you pay EC2 pricing for the instances. For Fargate, you pay per vCPU and memory consumed per second. A small production cluster (3x m5.large managed nodes) typically runs $150–200/month all-in. Use our EKS Cost Calculator to estimate based on your specific configuration.

What is the difference between EKS Fargate and managed node groups?

Managed node groups run EC2 instances that you pay for whether pods are scheduled or not — better for steady, predictable workloads. Fargate is serverless: you pay only for pod CPU and memory consumption with no node overhead, making it cost-effective for sporadic or variable workloads. Fargate has limitations: no DaemonSets, no privileged containers, and slightly higher per-unit cost than EC2 at high utilization. Use karpenter for a flexible middle ground on managed nodes.

How do I configure Kubernetes autoscaling on EKS?

EKS supports three autoscaling mechanisms: HPA (Horizontal Pod Autoscaler) scales pod replicas based on CPU/memory or custom metrics; VPA (Vertical Pod Autoscaler) adjusts resource requests over time; and Cluster Autoscaler or Karpenter scales node groups up and down. The most common production setup is HPA for pod scaling with Karpenter for node provisioning. Set HPA targetCPUUtilizationPercentage to 60–70% to leave headroom before the cluster autoscaler kicks in.

What is EKS VPC CNI and how does it affect networking performance?

The Amazon VPC CNI plugin assigns real VPC IP addresses to each pod, enabling native VPC routing without overlay networks. This gives low latency and high throughput but consumes IP addresses from your VPC subnets. For large clusters, IP exhaustion is a common issue — mitigate with ENABLE_PREFIX_DELEGATION=true to assign /28 CIDR prefixes to ENIs instead of individual IPs. Security Groups for Pods allows per-pod security group rules, but adds latency overhead from ENI attachment.

How do I reduce AWS EKS costs?

The most impactful cost reductions: (1) right-size pod resource requests — over-provisioning is the biggest waste driver; (2) use Spot instances for fault-tolerant workloads (60–90% cheaper than On-Demand); (3) enable Cluster Autoscaler to scale down unused nodes during off-peak hours; (4) use Savings Plans or Reserved Instances for baseline capacity; (5) move batch and scheduled workloads to Fargate to avoid always-on node costs. Use our Resource Analyzer and Cost Calculator to quantify each opportunity.

What is pod scheduling in Kubernetes and EKS?

The Kubernetes scheduler assigns pods to nodes based on resource availability, node selectors, affinity/anti-affinity rules, taints and tolerations, and topology spread constraints. Common scheduling failures on EKS include: insufficient CPU/memory (Pending pods), node taints without matching tolerations, and PodAntiAffinity preventing placement when there are fewer nodes than replicas. Use kubectl describe pod <name> to see scheduling failure reasons.
